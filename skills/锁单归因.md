# 锁单归因指标地图（Metrics Map）

## 1. 目标

用“锁单归因指标地图”把锁单表现拆成：规模 → 结构 → 驱动，并将每一天作为一个“样本”，在指定“样本范围”内与多个历史样本范围对比，评估当前周期的锁单表现处于历史什么水平、差异来自哪里。

## 2. 指标地图（从结果到原因）

```text
锁单
├─ 目标指标（结果）
├─ 结构指标（结构拆解）
│  ├─ 渠道结构
│  ├─ 漏斗结构
│  ├─ 用户路径结构
│  └─ 产品结构
└─ 驱动因素（解释变量）
   ├─ 价格驱动
   ├─ 活动驱动
   └─ 门店驱动
```

## 3. 样本与对比框架

### 3.1 样本定义

- 样本粒度：天（Day）。
- 样本集合：按“样本范围”筛选出的一组天。
  - 当前样本范围：本次要评估的周期（例如最近 7 天、自然周、活动期）。
  - 历史样本范围：用于对比的一个或多个历史周期（例如上周同周期、近 8 周同星期分布、去年同期、非活动基准期）。
- 业务周期样本范围：可直接使用 business_definition.json 的 time_periods（start~end 作为“业务窗口”，start~finish 作为“转化观察窗截止”）。

### 3.2 对比输出（建议）

对每个指标，在当前样本范围内做聚合（均值/中位数/分位数），并与每个历史样本范围对比：

- 绝对差（Δ）
- 相对差（Δ%）
- 历史分位（当前值在历史日样本分布中的 percentile）

### 3.3 时间口径（按指标选择时间字段）

- 锁单量：基于 lock_time（需过滤 lock_time 非空）
- 开票数 / 开票金额：基于 invoice_upload_time（需过滤 invoice_upload_time 非空）
- 交付数：基于 delivery_date（需过滤 delivery_date 非空）
- 小订数：基于 intention_payment_time（需过滤 intention_payment_time 非空）
- 外部线索下发相关指标：基于 Assign Time 年/月/日（仅限 assign_data）

### 3.4 数据源与可用字段（最小集）

- 订单/用户表：order_full_data.parquet
  - 核心时间：lock_time、invoice_upload_time、delivery_date、intention_payment_time、first_touch_time、first_test_drive_time、order_create_time/order_create_date
  - 核心维度：series、product_name、first_middle_channel_name、store_name、store_city、parent_region_name
  - 人（去重用）：owner_cell_phone
  - 价格：invoice_amount
  - 试驾：td_countd
- 外部线索表：assign_data.csv
  - 时间：Assign Time 年/月/日
  - 指标：下发线索数、下发线索当日试驾数、下发线索 7 日试驾数、下发线索 30日试驾数、下发线索 7 日锁单数、下发线索 30 日锁单数
  - 门店渠道：下发线索数 (门店)、下发线索当日锁单数 (门店)

## 4. 目标指标（Target Metrics，严格按 Schema 口径）

- 锁单量：order_number 计数（过滤 lock_time 非空；时间筛选基于 lock_time）
- 锁单人数（当前数据集口径）：owner_cell_phone 去重计数（过滤 lock_time 非空；且 owner_cell_phone 非空）
- 锁单率（外部线索口径，来自 assign_data）
  - 7 日锁单率（D7）：下发线索 7 日锁单数 / 下发线索数
  - 30 日锁单率（D30）：下发线索 30 日锁单数 / 下发线索数
  - 当日锁单率（D0，门店渠道可用）：下发线索当日锁单数 (门店) / 下发线索数 (门店)

## 5. 结构指标（Structure Metrics）

### 5.1 渠道结构

- 渠道维度（可用）：first_middle_channel_name（首次中间渠道名称）
- 分渠道锁单量：order_number 计数（过滤 lock_time 非空；按 lock_time 选样本）
- 分渠道开票量：order_number 计数（过滤 invoice_upload_time 非空；按 invoice_upload_time 选样本）
- 分渠道开票金额：invoice_amount 求和/平均（过滤 invoice_upload_time 非空）
- 外部线索分渠道（如 assign_data 提供渠道字段时）：下发线索数、D7/D30 锁单率

- 直接锁单（Direct）：X
- 归因锁单（Attributed）：X

辅助锁单（X）

- 首触辅助（First Touch）：X
- 过程助攻（Middle）：X
- 锁后触达（Post-Lock）：X

- 分来源渠道拆解（X）

### 5.2 漏斗结构

漏斗节点

- 线索 → 试驾 → 锁单 → 开票

漏斗效率

- 线索侧（assign_data，可直接按下发日观察）
  - 当日试驾率：下发线索当日试驾数 / 下发线索数
  - 7 日试驾率：下发线索 7 日试驾数 / 下发线索数
  - 30 日试驾率：下发线索 30日试驾数 / 下发线索数
  - 7 日锁单率：下发线索 7 日锁单数 / 下发线索数
  - 30 日锁单率：下发线索 30 日锁单数 / 下发线索数
- 订单侧（order_full_data，按事件时间观测）
  - 锁单量：lock_time 非空的 order_number 计数
  - 开票率（订单侧替代）：开票数 / 锁单量（开票数按 invoice_upload_time 非空计数）

### 5.3 用户路径结构

- 路径时间（可直接计算的替代口径）
  - 首触到试驾：first_test_drive_time - first_touch_time（均值/分位数）
  - 首触到锁单：lock_time - first_touch_time（过滤相关时间非空）
  - 试驾到锁单：lock_time - first_test_drive_time（过滤相关时间非空）
- 生命周期分布（建议）：按上述时间差做分箱分布，观察路径长短结构变化
- 触达渠道分布
- 人群过滤（可选）：age 建议限定在 [18, 80]

### 5.4 产品结构

- 基础车型：series（如 L6、LS6）
- 车型系列分组（series_group，来自 business_definition.json 的逻辑）
  - CM2：product_name LIKE '%新一代%' AND product_name LIKE '%LS6%'
  - CM1：product_name LIKE '%全新%' AND product_name LIKE '%LS6%'
  - CM0：product_name LIKE '%LS6%' AND product_name NOT LIKE '%全新%' AND product_name NOT LIKE '%新一代%'
  - DM1：product_name LIKE '%全新%' AND product_name LIKE '%L6%'
  - DM0：product_name LIKE '%L6%' AND product_name NOT LIKE '%全新%'
  - LS9：product_name LIKE '%LS9%'
  - LS7：product_name LIKE '%LS7%'
  - L7：product_name LIKE '%L7%'
  - 其他：ELSE
- 动力类型（product_type，需从 product_name 派生）
  - 增程：product_name matches 52|66
  - 纯电：product_name not matches 52|66
  - 过滤器要求：使用 matches / not matches 操作符，且正则为 52|66
- 车型系列映射（model_series_mapping）
  - LS6：CM0、CM1、CM2
  - L6：DM0、DM1
- 电池容量（battery_capacity，来自映射表）
  - 52kwh：新一代智己LS6 52 Max / 52 Max+ / 52 Pro、智己LS9 52 Ultra
  - 66kwh：新一代智己LS6 66 Max / 66 Max+、智己LS9 66 Ultra
  - 76kwh：新一代智己LS6 Max / Max+ / Pro
  - 103kwh：新一代智己LS6 Ultra / Pro Max

### 6.3 门店驱动

## 7. 评估“当前周期处于什么程度”（落地做法）

将每个指标按天产出日样本序列，并在样本范围上计算“当前周期聚合值”，再与历史日样本分布对比：

1. 选定当前样本范围与历史样本范围（可多个）。
2. 产出每一天的日指标（目标/结构/驱动同时产出）。
3. 当前周期聚合：对日样本做均值/中位数/分位数。
4. 历史对比：
   - 计算 Δ、Δ%
   - 计算历史分位：当前聚合值在历史日样本分布中的 percentile
5. 归因落点：当目标指标偏离时，优先用结构指标（渠道/漏斗/路径/产品）定位差异，再用驱动因素解释“为什么”。
