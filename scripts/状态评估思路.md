**一、对象与样本（把“每天”产品化成可评估的样本）**

- **样本粒度**：每天 = 1 个样本 `S(d)`；再定义一个“样本周期” `W = {d1..dk}`（如近7天/自然周/自然月）。
- **指标分层（建议按可控性+链路）**
  - **结果层**（订单表）：锁单数/开票数/退订/留存、在营门店数、店均锁单、ATP、CR5等
  - **过程层**（漏斗）：下发线索数 → 当日试驾 → 当日锁单、7/30日锁单率（分门店/直播/平台）
  - **结构层**（构成&归因）：车型 share（L6/LS6/LS9/REEV）、渠道Top、助攻渠道Top、用户分类占比
  - **健康层**（数据质量）：数据完整度、门店覆盖（下发门店数/在营门店数）、分母规模（避免小样本误报）

**二、历史对比的“多基线”设计（你说的“多个样本范围”核心）**
对每个当前周期 `W_cur`，同时对比多个历史样本周期 `W_hist^j`，用来消除不同噪声来源：

- **短期基线**：上一周期（前7天/前14天）——抓短期波动、策略变更影响
- **同周期基线**：近8–12周“同星期几集合”（如所有周六）——消除周内季节性
- **月内位置基线**：本月第N天 vs 历史各月第N天——消除月度节奏
- **节假日/活动基线**：按运营日历对齐（大促/车展/政策窗口）
- **分层基线**（用于诊断更关键）：按大区/城市级别/门店分层、按渠道/车型分层各自建基线（否则总体正常但结构异常会被掩盖）

落地形式：每个指标 m 都有一组基线分布 `B_m^j`（来自不同 `W_hist^j`），当前周期得到一组评分 `score_m^j`，最后做融合（例如取最异常的、或加权平均）。

**三、统计分布评估（把“表现到什么程度”量化成可解释的分布位置）**
把指标按类型选择合适的“分布评估器”，输出统一的：分位数/尾部概率/异常分数。

- **计数类**（锁单数、开票数、下发线索数、有效试驾数）
  - 用 **分位数位置**（p10/p50/p90）+ **稳健Z分数**（建议对数变换 `log(x+1)` 后用 MAD）
  - 如要更严谨可用 Poisson/负二项的预测分布做尾部概率 `p = P(X>=x)`
- **比率类**（当日试驾率、当日锁单率、7/30日锁单率）
  - 必须带上分母（线索数/门店线索数），用 **Beta-Binomial**（或至少 Wilson 区间）给出“这次观测是否超出随机波动”
  - 产出：当前比率在历史下的 **后验预测分位数/尾部概率**
- **构成/份额类**（share_ls6、渠道Top占比、用户分类占比）
  - 单个份额：分位数位置即可
  - 整体结构：用 **Jensen–Shannon divergence（JS散度）** 或 TopK 变化幅度，判断“结构漂移”
- **集中度类**（CR5门店销量集中度、CR5门店城市集中度）
  - 作为时间序列指标：看其在历史分布的位置 + 是否持续上行（EWMA更稳）
- **金额类**（整体ATP）
  - 强烈建议用分位数 + 稳健Z（ATP常有长尾）

**统一输出字段（建议每个指标都产出）**

- `value`、`baseline_median`、`pctl`（当前值在历史分布的百分位）、`tail_p`、`z_robust`、`direction`（偏高/偏低）、`denominator`（若有）

**四、告警与评分（把分布位置变成“红黄绿”和可追踪的稳定信号）**

- **单指标告警**（示例规则，实际可按业务容忍度调）
  - **红**：落在历史 p≤1% 或 p≥99% 且连续≥2天（或 tail_p≤0.01）
  - **黄**：p≤5% 或 p≥95%，或单日红但分母很小则降级
  - **信息**：结构漂移（JS散度超过阈值）但结果层未受损
- **周期评估**（对 `W_cur` 而非单天）
  - 对比 `W_hist^j`：做“均值/中位数差异 + 分布检验”（Mann–Whitney/KS二选一即可）
  - 产出“本周期较历史的提升/下滑幅度 + 置信度”
- **总健康分**：把核心链路分成几块（结果/漏斗/结构/数据质量），每块取最异常的几个指标加权；数据质量不过线直接“降权或不评估”（避免脏数触发误报）。

**五、监控输出 → 诊断归因（报警后 5 分钟能定位“哪里坏了”）**

- **一页总览（每天/每周期）**
  - 本日/本周期：锁单、开票、线索、试驾、关键转化率的分位数位置（相对多基线）
  - Top 异常指标（按异常分数排序）+ 是否“持续异常”
  - 数据质量卡片：数据完整度、门店覆盖、关键分母
- **诊断下钻顺序（固定套路，减少人肉分析成本）**
  1. **结果层异常**（如锁单/开票）先拆：按大区→城市→门店，做贡献度（哪几个区域拉低最多）
  2. 同时看 **过程层**：是线索掉了、试驾率掉了、还是锁单率掉了（分子分母一起看）
  3. 再看 **结构层**：车型 share / 渠道 mix 是否漂移（例如自然客流占比异常上升但总体锁单下降，可能意味着渠道质量变化）
  4. 最后看 **归因助攻**：跨渠道助攻Top变化（定位“前链路哪里断了”）
- **贡献度计算（够用且易实现）**
  - 计数类：`Δ总量 = Σ(当前-基线)`，按维度分组做 waterfall 排序
  - 比率类：用“分母变化 + 比率变化”的两因子拆解（先看分母，再看比率）

这套框架的好处是：你已经有了每天的指标 JSON 输出；接下来只要把“当前周期 vs 多基线周期”的分布评估、评分、告警、下钻贡献度串起来，就能稳定回答“当前样本周期的锁单表现处在历史什么水平、异常来自哪里、是否值得介入”。
